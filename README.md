# Swift .NET Packages

A monorepo of NuGet packages that bring popular Swift libraries to .NET. Each package contains auto-generated C# bindings and is validated on-device through a .NET for iOS simulator test app.

Bindings are generated by [swift-dotnet-bindings](https://github.com/justinwojo/swift-dotnet-bindings) and target .NET 10.0 on Apple platforms.

## Libraries

| Package | Description | Version | Mode |
|---------|-------------|---------|------|
| `Swift.Nuke` | High-performance image loading and caching | 12.8.0 | Source |
| `Swift.Lottie` | Lottie animation rendering | 4.6.0 | Source |
| `Swift.BlinkID` | Identity document scanning (Microblink) | 7.6.2 | Binary |
| `Swift.BlinkIDUX` | BlinkID scanning UI components | 7.6.2 | Source |
| `Swift.Stripe.*` | Stripe payments SDK (11 products) | 25.6.2 | Source |

Stripe products include StripeCore, StripePayments, StripePaymentSheet, StripeApplePay, StripeIdentity, StripeIssuing, StripeCardScan, StripeFinancialConnections, StripeConnect, and more.

## How It Works

Each library follows the same workflow:

1. **Build xcframeworks** — fetch the Swift library via SPM and build device + simulator xcframeworks
2. **Generate bindings** — `dotnet build` invokes the Swift Bindings SDK, which reads the xcframework and emits C# bindings + a Swift interop wrapper
3. **Validate on simulator** — a .NET for iOS test app exercises the bindings on an iOS Simulator, verifying that Swift APIs are callable end-to-end from C#

## Quick Start

```bash
# Build a library's xcframework
cd libraries/Nuke && ./build-xcframework.sh

# Generate bindings and compile (SDK handles generation automatically)
dotnet build Swift.Nuke.csproj

# Run simulator tests
cd tests/Nuke.SimTests
./build-testapp.sh
xcrun simctl boot <device-udid>
./validate-sim.sh 15
```
