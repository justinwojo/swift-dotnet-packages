# SwiftBindings.Mappedin

Native Swift interop bindings for [Mappedin](https://www.mappedin.com/), the indoor mapping and wayfinding SDK. These are not Objective-C proxy wrappers — they use .NET 10's native Swift interop for direct, high-performance calls into Swift APIs from C#.

## Installation

```
dotnet add package SwiftBindings.Mappedin
```

## Requirements

- .NET 10.0+
- iOS 15.0+
- macOS host for development
- A Mappedin developer account and API credentials for live data

## Usage

### Position and coordinates

```csharp
using Mappedin;

var coords = new MPICoordinates(
    latitude: 43.6532,
    longitude: -79.3832,
    accuracy: 10.0,
    floorLevel: 2);

var position = new MPIPosition(
    timestamp: 1000.0,
    coords: coords,
    type: "gps",
    annotation: "user",
    bearing: 90.0);
```

### Map view

```csharp
var frame = new CoreGraphics.CGRect(0, 0, 375, 667);
var mapView = new MPIMapView(frame);

// Authorization headers when loading map data
var auth = new MPIHeader("Authorization", "Bearer your_token_here");
```

See [Mappedin's iOS documentation](https://docs.mappedin.com/) for venue loading, navigation, search, and event handling.

## Caveats

The Mappedin xcframework is licensed by Mappedin and not open source — it must be provisioned out-of-band. The NuGet package ships the binding metadata; consumers must obtain the Mappedin SDK directly from Mappedin and follow their licensing terms.

## How It Works

These bindings are auto-generated by [Swift Dotnet Bindings](https://github.com/justinwojo/swift-dotnet-bindings) using .NET 10's native Swift interop via the [SwiftBindings SDK](https://www.nuget.org/packages/SwiftBindings.Sdk).
