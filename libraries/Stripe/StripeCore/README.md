# SwiftBindings.Stripe.Core

Native Swift interop bindings for `StripeCore` — the foundational module of the [Stripe iOS SDK](https://github.com/stripe/stripe-ios). It hosts the API client (`STPAPIClient`), networking primitives, the `STPAppInfo` partner-info struct, error types, and shared cross-module types that every other Stripe package depends on. These are not Objective-C proxy wrappers — they use .NET 10's native Swift interop for direct, high-performance calls into Swift APIs from C#.

## Installation

```
dotnet add package SwiftBindings.Stripe.Core
```

Most apps don't take a direct dependency on `StripeCore` — they pull in [`SwiftBindings.Stripe.Payments`](https://www.nuget.org/packages/SwiftBindings.Stripe.Payments) or [`SwiftBindings.Stripe.PaymentSheet`](https://www.nuget.org/packages/SwiftBindings.Stripe.PaymentSheet), which transitively depend on this package. Add it directly only if you need to interact with `STPAPIClient` / `STPAppInfo` standalone.

## Requirements

- .NET 10.0+
- iOS 15.0+
- macOS host for development

## Usage

```csharp
using StripeCore;

// Configure the API client with your publishable key
var client = new STPAPIClient("pk_test_...");

// Identify your integration (optional, for partner attribution)
var appInfo = new STPAppInfo("MyApp", "partner_123", "1.0.0", "https://example.com");
```
