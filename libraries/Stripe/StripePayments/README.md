# SwiftBindings.Stripe.Payments

Native Swift interop bindings for `StripePayments` — the module of the [Stripe iOS SDK](https://github.com/stripe/stripe-ios) that handles `PaymentIntent`, `SetupIntent`, `PaymentMethod`, and the underlying confirm/handle-action flows. Use this package when you need fine-grained control over payment confirmation; if you want a drop-in UI, use [`SwiftBindings.Stripe.PaymentSheet`](https://www.nuget.org/packages/SwiftBindings.Stripe.PaymentSheet) instead. These are not Objective-C proxy wrappers — they use .NET 10's native Swift interop for direct, high-performance calls into Swift APIs from C#.

## Installation

```
dotnet add package SwiftBindings.Stripe.Payments
```

## Requirements

- .NET 10.0+
- iOS 15.0+
- macOS host for development

## Usage

### Build a payment method

```csharp
using StripePayments;

var card = new STPPaymentMethodCardParams();
var billing = new STPPaymentMethodBillingDetails();
var address = new STPPaymentMethodAddress();

var pmParams = new STPPaymentMethodParams();
```

### Confirm a PaymentIntent

```csharp
var confirm = new STPPaymentIntentConfirmParams("pi_test_secret_123");
// configure confirm.PaymentMethodParams etc., then call STPPaymentHandler.SharedHandler
```

### SetupIntent flow

```csharp
var setup = new STPSetupIntentConfirmParams("seti_test_secret_123");
```
